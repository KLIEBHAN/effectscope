# Product

## Register

product

## Users

React learners with basic component and state knowledge, including bootcamp and
university students, plus mentors explaining asynchronous effect behavior. They
use EffectScope during a focused learning workflow to predict behavior, inspect
runtime evidence, choose a repair, and prove the result.

## Product Purpose

EffectScope makes temporal React effect behavior observable. It runs checked-in,
instrumented React scenarios, records normalized lifecycle and asynchronous
events, and evaluates invariants deterministically. Optional GPT-5.6 coaching
explains the learner's mental-model gap from validated evidence without becoming
a competing source of technical truth. Success means learners can predict a bug,
identify its cause in the trace, choose the smallest effective repair, and
distinguish repaired behavior from the original failure.

## Brand Personality

Precise, evidence-led, and instructional. The interface should feel like a
trustworthy technical instrument: direct enough for experienced developers,
clear enough for learners, and calm when showing failures.

## Anti-references

- Generic AI chat interfaces that make model prose look authoritative.
- Decorative SaaS dashboards whose cards and metrics obscure the learning loop.
- Error-heavy developer tools that use red for every unsuccessful technical
  state, including correct learner reasoning.
- Simulators that imply invented events are equivalent to observed runtime
  behavior.

## Design Principles

1. Separate learner performance from runtime invariant status.
2. Make execution evidence more prominent than model explanation.
3. Preserve the predictable Predict, Run, Observe, Explain, Repair, Prove loop.
4. Use concise copy that explains both situation and consequence.
5. Prefer familiar controls and explicit state labels over decorative novelty.

## Accessibility & Inclusion

Maintain keyboard operation, visible focus, semantic controls, permanent atomic
status announcements, reduced-motion behavior, meaningful non-color state cues,
and responsive operation down to 390 pixels. Text and control borders must retain
sufficient contrast in the dark interface.
